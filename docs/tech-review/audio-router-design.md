# 선택적 오디오 라우터 설계 문서

본 문서는 WebRTC SFU 위에서 동작하는 선택적 오디오 라우팅 미들웨어의 아키텍처와 인터페이스를 정의한다. 학부 캡스톤 범위에서는 구현 외이며, 카풀링 내부 개발팀의 통합 작업 시 참고 자료로 작성되었다.

## 모듈 목적

기본 SFU 구조에서는 한 세션에 참여한 모든 참여자가 동일한 미디어 스트림을 수신한다. 그러나 카풀링 멘토링 서비스에서는 참여자마다 다른 오디오를 전달해야 하는 상황이 존재한다. 본 모듈은 SFU 위에서 동작하며, 누가 어떤 오디오 스트림을 받을지 동적으로 제어한다.

대표적인 차별적 라우팅 시나리오는 다음과 같다.

1. **골드챗 1:1 전환**: 50,000원 이상 결제한 멘티만 멘토의 비공개 음성 채널을 수신
2. **음소거 멘티**: 특정 멘티의 음성을 다른 멘티들에게 전달하지 않음
3. **귓속말 기능**: 멘토가 특정 멘티에게만 음성 메시지를 전송
4. **품질 등급 차등**: 멘티의 결제 등급에 따라 음성 비트레이트를 차등 전송

## 시스템 위치

본 모듈은 SFU와 클라이언트 사이에서 라우팅 정책을 적용하는 미들웨어 계층에 위치한다.

```
┌─────────────────┐
│ Mediasoup SFU   │   모든 Producer / Consumer 이벤트
└────────┬────────┘
         │
         ▼
┌─────────────────────┐
│ 오디오 라우터 미들웨어 │  ◀── 본 모듈
│ (라우팅 정책 엔진)    │
└────────┬────────────┘
         │ 정책 적용 후 선택적 전달
         ▼
┌─────────────────┐
│ 각 멘티 클라이언트 │  (참여자마다 다른 스트림 수신 가능)
└─────────────────┘
```

## 라우팅 규칙 구조

라우팅 규칙은 "어떤 송신자(Producer)의 스트림을 어떤 수신자(Consumer)에게 전달할 것인가"를 정의한다.

```typescript
interface RoutingRule {
  ruleId: string;
  sessionId: string;
  producerId: string;        // 송신자 (보통 멘토)
  allowedConsumers: string[]; // 수신 허용 사용자 ID 목록
  bitrate?: number;           // 차등 비트레이트 (bps), 미지정 시 기본값
  exclusive?: boolean;        // true면 allowedConsumers 외 전원 차단 (1:1 전환)
  priority: number;           // 규칙 충돌 시 우선순위
  expiresAt?: string;         // 규칙 만료 시점 (ISO 8601), 미지정 시 세션 종료까지
}
```

## 라우팅 정책 동작 방식

1. SFU에서 새로운 Consumer 생성 요청이 발생하면, 라우터가 먼저 해당 세션의 활성 규칙을 조회한다.
2. 요청한 사용자가 `allowedConsumers`에 포함되는지 검사한다.
3. `exclusive`가 true인 규칙이 있으면, 허용 목록 외 사용자의 기존 Consumer를 일시 중단한다.
4. 통과 시 `bitrate` 설정에 따라 스트림 품질을 조정하여 전달한다.
5. 규칙이 충돌하면 `priority`가 높은 규칙을 우선 적용한다.

## 인터페이스 정의

### REST API

| 메소드 | URL | 설명 |
|--------|-----|------|
| POST | `/api/audio-router/v1/rules` | 라우팅 규칙 추가 |
| GET | `/api/audio-router/v1/sessions/:sessionId/rules` | 세션 내 활성 규칙 조회 |
| PATCH | `/api/audio-router/v1/rules/:ruleId` | 규칙 수정 |
| DELETE | `/api/audio-router/v1/rules/:ruleId` | 규칙 제거 |

### 질문 큐 매니저와의 연계 이벤트

오디오 라우터는 질문 큐 매니저의 슈퍼챗 처리 이벤트를 구독하여 규칙을 자동 생성·제거한다.

| 이벤트 | 동작 |
|--------|------|
| `gold_chat:accepted` | 해당 멘티와 멘토만 수신하는 `exclusive` 라우팅 규칙 자동 생성 |
| `gold_chat:ended` | 해당 1:1 라우팅 규칙 자동 제거, 전체 세션 오디오 복귀 |
| `mentee:muted` | 해당 멘티의 Producer를 전체 수신 목록에서 제외 |

## 골드챗 1:1 전환 시나리오 예시

```
1. 멘티가 골드챗(50,000원) 결제 → 질문 큐 매니저에 등록
2. 멘토가 1:1 수락 → gold_chat:accepted 이벤트 발행
3. 오디오 라우터가 exclusive 규칙 생성
   { producerId: 멘토, allowedConsumers: [해당 멘티], exclusive: true }
4. 멘토 음성이 해당 멘티에게만 전달, 나머지 멘티는 대기
5. 1:1 종료 → gold_chat:ended → 규칙 제거 → 전체 세션 복귀
```

## 카풀링 서비스 통합 시 고려사항

1. **결제 연동**: 골드챗 1:1 전환 규칙은 결제 캡처(최종 승인) 시점에 생성되어야 하며, 결제 실패 시 규칙이 생성되지 않도록 해야 한다.
2. **세션 만료 처리**: 멘토링 세션이 종료되면 해당 세션의 모든 라우팅 규칙을 자동 정리하여 메모리 누수와 오작동을 방지한다.
3. **음성 품질 하한**: 차등 비트레이트를 적용하더라도 최소 16kbps Opus 수준은 보장하여, 저등급 사용자도 알아들을 수 있는 품질을 유지한다.
4. **규칙 적용 로깅**: 누가 언제 어떤 오디오를 수신했는지에 대한 이력을 별도 로그로 보관하여, 결제 분쟁 발생 시 증빙 자료로 활용한다.
5. **지연 최소화**: 라우팅 정책 평가가 미디어 전달 경로에 추가 지연을 유발하지 않도록, 규칙 조회는 인메모리 캐시 기반으로 처리한다.
