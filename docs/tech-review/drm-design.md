# DRM / 캡처 방지 모듈 설계 문서

본 문서는 카풀링 인터랙티브 멘토링 시스템에서 멘토링 콘텐츠의 무단 캡처·복제·재배포를 방지하기 위한 DRM(Digital Rights Management) 및 캡처 방지 모듈의 아키텍처와 인터페이스를 정의한다. 학부 캡스톤 범위에서는 정식 DRM 라이선스 발급이 전제되어야 하므로 직접 구현 대상에서 제외하였으며, 본 문서는 카풀링 내부 개발팀의 통합 작업 시 참고 자료로 작성되었다.

## 모듈 목적

멘토링 세션에서 멘토가 제공하는 강의 영상, 슬라이드, 음성은 멘토의 전문성이 담긴 유료 콘텐츠다. 본 모듈은 다음을 보호한다.

1. 실시간 스트리밍 영상·음성의 무단 화면 녹화 및 화면 캡처 방지
2. 멘토가 업로드한 슬라이드 자료의 무단 다운로드 방지
3. 멘토링 종료 후 제공되는 '다시 듣기'(VOD) 콘텐츠의 외부 유출 방지
4. 골드챗 1:1 비공개 멘토링 내용의 보호

## 보호 대상과 위협 모델

| 보호 대상 | 주요 위협 | 대응 수단 |
|-----------|-----------|-----------|
| 실시간 스트림 | 화면 녹화 도구, 외부 캡처 장비 | EME 암호화, 동적 워터마크 |
| 슬라이드 자료 | 다운로드, 스크린샷 | 전용 뷰어, 우클릭·단축키 차단 |
| VOD 다시듣기 | 파일 추출, 재배포 | DRM 패키징, 토큰 기반 재생 |
| 1:1 비공개 멘토링 | 제3자 청취, 녹음 | 종단간 암호화(E2EE) |

## 기술 선택

### EME (Encrypted Media Extensions)

EME는 W3C 표준 브라우저 API로, 암호화된 미디어 스트림을 브라우저가 직접 복호화·재생할 수 있도록 한다. 콘텐츠 키는 라이선스 서버를 통해서만 발급되므로, 키 없이는 스트림을 재생하거나 추출할 수 없다. 본 시스템은 EME 기반 DRM을 핵심 보호 수단으로 채택한다.

### DRM 시스템 비교

| 항목 | Widevine | FairPlay | PlayReady |
|------|----------|----------|-----------|
| 제공사 | Google | Apple | Microsoft |
| 주요 플랫폼 | Chrome, Android, 스마트TV | Safari, iOS, macOS | Edge, Windows |
| 보안 등급 | L1 / L2 / L3 | 단일 등급 | SL150 / SL2000 / SL3000 |
| 라이선스 비용 | 무료(사용 등록 필요) | Apple 개발자 등록 필요 | 유료 라이선스 |
| 모바일 대응 | Android 강점 | iOS 강점 | 제한적 |

카풀링은 모바일 앱과 웹을 동시에 지원해야 하므로, **Android·Chrome 계열은 Widevine, iOS·Safari 계열은 FairPlay를 병행**하는 멀티 DRM 전략을 권장한다.

## 시스템 아키텍처

```
┌──────────────┐       ┌─────────────────┐       ┌──────────────┐
│ 멘토 송출     │──────▶│ 암호화 서비스    │──────▶│ 멘티 클라이언트│
│ (영상/음성)   │       │ (Packager/KMS)  │       │ (EME 재생)    │
└──────────────┘       └────────┬────────┘       └──────┬───────┘
                                │                        │
                                ▼                        ▼
                       ┌─────────────────┐       ┌──────────────┐
                       │ 라이선스 서버    │◀──────│ 라이선스 요청 │
                       │ (Widevine/      │ 키요청 │ (재생 시점)   │
                       │  FairPlay)      │       └──────────────┘
                       └─────────────────┘
```

처리 흐름:
1. 멘토가 송출한 스트림을 암호화 서비스가 콘텐츠 키로 암호화
2. 암호화된 스트림은 일반 CDN으로 전송 (키 없이는 재생 불가)
3. 멘티가 재생을 시작하면 클라이언트가 라이선스 서버에 키 요청
4. 라이선스 서버가 사용자 권한(결제·세션 참여 여부)을 검증 후 키 발급
5. 클라이언트의 EME가 키로 복호화하여 재생

## 학부 캡스톤 범위에서 구현 가능한 대안

정식 DRM 라이선스 발급에는 별도 등록 절차와 비용·기간이 필요하므로, 캡스톤 범위에서는 다음 경량 보호 기법을 우선 적용할 수 있다. 완전한 차단은 불가능하나 일반 사용자 수준의 무단 복제를 효과적으로 억제한다.

### 1. 동적 워터마크

스트리밍 영상과 슬라이드에 시청자의 사용자 ID와 시청 시각을 반투명 텍스트로 실시간 합성한다. 무단 캡처·녹화가 발생하더라도 유출 경로를 추적할 수 있어, 사후 책임 추궁과 심리적 억제 효과를 동시에 제공한다.

### 2. 화면 녹화 감지

브라우저의 Page Visibility API와 Screen Capture API를 활용해 화면 공유·녹화 행위를 감지한다. 녹화가 감지되면 경고 메시지를 표시하거나 스트림을 일시 중단할 수 있다.

### 3. 우클릭 및 단축키 차단

`contextmenu` 이벤트와 `keydown` 이벤트를 가로채 일반적인 저장·캡처 단축키(Ctrl+S, PrintScreen, F12 개발자 도구 등)를 차단한다. 우회가 가능하나 일반 사용자에게는 효과적인 1차 방어선이다.

### 4. 전용 플레이어 재생

VOD 다시듣기 콘텐츠는 직접 다운로드 가능한 파일 형태로 제공하지 않고, 플랫폼 내 전용 플레이어에서 토큰 기반으로만 재생되도록 제한한다. 재생 토큰은 만료 시간을 두어 링크 공유를 무력화한다.

## 인터페이스 정의

### REST API

| 메소드 | URL | 설명 |
|--------|-----|------|
| POST | `/api/drm/v1/keys` | 콘텐츠 키 발급 (멘토 송출 시작 시) |
| POST | `/api/drm/v1/license` | 재생 라이선스 요청 (멘티 재생 시) |
| GET | `/api/drm/v1/watermark/:sessionId/:userId` | 동적 워터마크 정보 조회 |
| POST | `/api/drm/v1/capture-report` | 화면 녹화 감지 이벤트 보고 |

### 라이선스 요청 검증 항목

라이선스 서버는 키 발급 전 다음을 검증한다.

- 요청 사용자가 해당 세션의 정상 참여자인지
- 비공개(골드챗) 콘텐츠라면 해당 권한을 결제했는지
- 세션이 아직 유효한 상태인지(종료된 세션의 키 발급 차단)

## 카풀링 서비스 통합 시 고려사항

1. **라이선스 발급 절차**: Widevine은 Google에 사용 등록이 필요하며(통상 1~2주 소요), FairPlay는 Apple Developer 인증서 발급이 필요하다. 통합 일정에 이 기간을 반영해야 한다.
2. **키 관리 서버 분리**: 암호화된 스트림은 일반 CDN으로 전송 가능하나, 콘텐츠 키를 관리하는 KMS와 라이선스 서버는 반드시 별도 보안 영역에 배치한다.
3. **모바일 앱 통합**: 웹뷰가 아닌 네이티브 플레이어(Android ExoPlayer, iOS AVPlayer) 사용을 권장한다.
4. **법적 고지**: 화면 녹화 감지 등 일부 기능은 사용자 사전 동의가 필요하므로 서비스 약관과 개인정보 처리방침에 명시해야 한다.
5. **성능 영향**: 실시간 암호화는 지연(latency)을 유발할 수 있으므로, WebRTC SFU 모듈과의 연동 시 암호화 단계가 200ms 이하 목표 지연을 초과하지 않도록 검증이 필요하다.
