# 프론트엔드 와이어프레임 및 화면 설계

본 문서는 카풀링 인터랙티브 멘토링 시스템의 프론트엔드 화면 설계를 정리한다. 통합 데모는 멘토 화면, 멘티 화면, 음성 명령 인식 화면 세 가지로 구성되며, 각 화면은 카풀링 서비스의 핵심 사용자 경험을 보여주는 데 초점을 둔다.

## 설계 원칙

### 비대칭 화면 구성

기존의 일반적인 라이브 스트리밍 플랫폼은 멘토와 멘티가 동일한 인터페이스를 본다. 그러나 카풀링은 멘토가 운전 중일 수 있다는 특수한 상황을 전제로 하므로, 두 사용자 그룹에 서로 다른 화면을 제공한다.

- **멘토 화면**: 정보 밀도 최소화, 큰 버튼, 고대비, 큐레이션된 핵심 메시지만 표시
- **멘티 화면**: 정보 밀도 일반, 모든 채팅 흐름 표시, 슈퍼챗 입력 UI 포함

### 운전자 친화적 UX

NHTSA 가이드라인에 따른 시선 분산 최소화 원칙을 적용한다.

- 글랜스 최적화: 한 번에 인지 가능한 정보 단위로 화면 구성
- 음성 우선: 시각 대신 청각으로 정보 전달 (TTS)
- 대형 터치 영역: 운전 중 정밀 조작이 어려운 점을 고려한 버튼 크기

│  │                                            │ │
│  └────────────────────────────────────────────┘ │

│  │ └────────────────────────────────────────┘ │ │
│  └────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────┘

### 핵심 차별 요소

1. **드라이브 모드 토글**: 주행 상태 시뮬레이션을 위한 수동 토글. 실제 서비스에서는 GPS와 가속도계 자동 감지로 대체됨.
2. **화면 차단 표시**: 드라이브 모드 ON 시 영상/슬라이드 영역이 차단되어 운전 주의 분산 방지.
3. **큐레이션된 메시지**: 채팅 서버에서 `chat:curated` 이벤트로 받은 메시지만 표시. 일반 멘티 화면과 비대칭 정보 구성.
4. **대형 버튼**: 슬라이드 전환 버튼이 화면 폭의 절반씩 차지하여 운전 중에도 시선 이동 없이 조작 가능.

## 멘티 화면 (`/mentee`)

### 레이아웃 구조

┌──────────────────────────────────────────────────┐
│ 멘티 화면           [슬라이드 -] [슬라이드 +]    │

├──────────────────────────────────────────────────┤
│  ┌────────────────────────────────────────────┐ │
│  │ 📊 슬라이드 3                              │ │

│  └────────────────────────────────────────────┘ │
├──────────────────────────────────────────────────┤
│                                                  │
│  ┌──────────────────────────────────────┐       │
│  │ 김멘토                               │       │

│  │ 슬라이드 3                            │       │
│  └──────────────────────────────────────┘       │
│                                                  │
│         ┌──────────────────────────────────────┐ │
│         │ 테스트멘티                            │ │
│         │ 안녕하세요?                           │ │

│         └──────────────────────────────────────┘ │
│                                                  │
├──────────────────────────────────────────────────┤
│ [슈퍼챗 등급 선택 시 표시 영역]                  │
│  Bronze (5천)  Silver (2만)  Gold (5만)         │
├──────────────────────────────────────────────────┤
│ [입력창................] [💰] [전송]            │
└──────────────────────────────────────────────────┘

### 핵심 차별 요소

1. **슬라이드 컨텍스트 자동 부여**: 메시지를 보낼 때 현재 슬라이드 번호가 자동으로 포함되어 멘토가 어느 슬라이드에 대한 메시지인지 추적 가능.
2. **슈퍼챗 등급 선택**: 💰 버튼 클릭 시 브론즈/실버/골드 등급 선택 UI가 나타나며, 등급별 가격이 명시됨.
3. **양방향 표시**: 본인이 보낸 메시지는 우측 정렬, 타인의 메시지는 좌측 정렬되어 채팅 흐름이 직관적.
4. **백엔드 이중 등록**: 슈퍼챗은 채팅 서버(3001)와 질문 큐 서버(3002) 양쪽에 동시 등록되어 질문 큐 우선순위 처리 가능.

│  ┌────────────────────────────────────────────┐ │

│  │ "오늘 날씨가 좋네요"                        │ │
│  │ 호출어/명령어 없음 (강의 흐름 보호)         │ │
│  └────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────┘

### 핵심 차별 요소

1. **호출어 기반 인식**: "카풀링"이라는 호출어가 발화에 포함되어야만 명령으로 해석. 멘토가 강의 중 자연스럽게 한 말이 명령으로 오인식되는 것을 방지.
2. **음성 확인 루프**: 명령어 인식 후 즉시 실행하지 않고 TTS로 "진행할까요?"라고 되묻고, 사용자의 명시적 확인 후 실행. 운전 중 잘못된 조작을 방지하는 핵심 안전 장치.
3. **인식 기록 시각화**: 호출어 없는 발화도 로그에 표시되어, 시스템이 어떻게 발화를 분류하고 있는지 투명하게 보여줌. 디버깅과 시연에 모두 유용.
4. **시각적 상태 표시**: 듣는 중일 때는 빨간색 깜빡임, 대기 중일 때는 흰색 정적 표시로 사용자가 시스템 상태를 한눈에 파악 가능.

## 통합 시연 흐름

세 화면을 활용한 발표 시연 시나리오는 다음과 같다.

1. 일반 창에 멘토 화면을 띄우고 드라이브 모드 ON 설정
2. 시크릿 창에 멘티 화면을 띄우고 닉네임 설정 후 입장
3. 멘티 화면에서 일반 채팅 메시지 전송 → 멘토 화면 큐레이션 통과 확인
4. 멘티 화면에서 골드 슈퍼챗 전송 → 멘토 화면 강조 표시 + 질문 큐 서버 등록 확인
5. 멘토 화면을 음성 명령 인식 화면으로 전환 (실제 서비스에서는 통합되어 있음)
6. "카풀링, 다음 슬라이드" 발화 → 확인 루프 → 실행
7. "오늘 발표 잘 되고 있죠?" 발화 → 호출어 없음 → 강의 흐름 보호

## 향후 통합 시 고려사항

본 와이어프레임은 학부 캡스톤 범위의 프로토타입이며, 실제 카풀링 앱 통합 시 다음 사항이 추가로 고려되어야 한다.

1. **반응형 디자인**: 모바일/태블릿/차량 디스플레이 등 다양한 화면 크기 대응
2. **다크 모드**: 야간 주행 환경에 적합한 색 구성 자동 전환
3. **접근성**: 시각 장애 사용자를 위한 스크린리더 호환성, 청각 장애 사용자를 위한 시각 알림
4. **국제화**: 호출어와 명령어의 다국어 지원 (영어, 일본어 등)
