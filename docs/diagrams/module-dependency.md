## 모듈별 의존성 상세

### 라이브 채팅 모듈

- **의존 대상:** Redis (Pub/Sub), Question Queue (슈퍼챗 메시지 동기화)
- **제공 인터페이스:** WebSocket 이벤트 (chat:join, chat:message, chat:broadcast 등)
- **소비처:** Frontend (Mentor UI / Mentee UI), Question Queue

### 질문 큐 매니저

- **의존 대상:** Redis (Sorted Set 우선순위 큐), Voice Command (TTS 트리거)
- **제공 인터페이스:** REST API (/api/questions), Redis Pub/Sub 채널
- **소비처:** Live Chat, Voice Command, Frontend

### 음성 명령 인식 모듈

- **의존 대상:** Question Queue (다음 질문 호출), 슬라이드 상태
- **제공 인터페이스:** WebSocket 이벤트 (voice:command 등)
- **소비처:** Frontend (Mentor UI)

### WebRTC SFU 서버 (설계 외)

- **의존 대상:** Audio Router, Live Chat (메타데이터 동기화)
- **인터페이스 정의 위치:** `docs/tech-review/webrtc-sfu-design.md` (예정)

### 선택적 오디오 라우터 (설계 외)

- **의존 대상:** WebRTC SFU
- **인터페이스 정의 위치:** `docs/tech-review/audio-router-design.md` (예정)

### DRM/캡처 방지 모듈 (설계 외)

- **의존 대상:** WebRTC SFU (스트림 암호화)
- **인터페이스 정의 위치:** `docs/tech-review/drm-design.md` (예정)

## 데이터 흐름 핵심 포인트

1. **Redis Pub/Sub**이 모든 백엔드 모듈의 메시지 허브 역할
2. **세션 ID**가 모든 모듈에서 공유되는 핵심 키
3. **슬라이드 번호**가 채팅과 질문 큐 모두에 컨텍스트로 포함됨
4. 프론트엔드는 모듈별로 독립된 WebSocket 연결을 유지함
