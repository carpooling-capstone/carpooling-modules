# 사용 시나리오 정리

본 문서는 카풀링 인터랙티브 멘토링 시스템의 핵심 사용 시나리오를 정리한 것이다. 세 가지 시나리오는 통합 데모의 시연 흐름 설계에 사용된다.

## 시나리오 1: 일반 멘토링

### 참여자

- 멘토 1명 (운전 중)
- 멘티 다수 (자택, 사무실 등)

### 흐름

1. 멘토가 카풀링 앱에서 멘토링 세션 생성
2. 멘티들이 세션 코드로 입장
3. 멘토가 사전 업로드한 슬라이드 표시
4. 멘토가 음성으로 강의 진행 (드라이브 모드 활성화 상태)
5. 멘토가 음성 명령으로 슬라이드 전환 ("카풀링, 다음")
6. 시스템이 확인 음성 출력 ("다음 슬라이드로 넘어갈까요?")
7. 멘토가 "응" 또는 침묵으로 확정 시 전환 실행

### 관련 모듈

- 음성 명령 인식 모듈
- 라이브 채팅 모듈 (멘티 반응 표시)

## 시나리오 2: 슈퍼챗 응대

### 흐름

1. 멘토가 강의 중 특정 슬라이드를 표시 중
2. 멘티가 5,000원 브론즈 챗으로 질문 등록
3. 질문 큐 매니저가 슬라이드 컨텍스트와 함께 질문 저장
4. 일정 시간 후 다른 멘티가 같은 슬라이드에 질문 추가
5. 같은 슬라이드 질문이 자동 그룹핑됨
6. 멘토가 "다음 질문" 음성 명령 실행
7. TTS로 멘티 질문이 음성 출력됨
8. 멘토가 운전대에서 손을 떼지 않고 음성으로 응답
9. 응답 완료 후 결제 캡처(최종 처리)

### 관련 모듈

- 질문 큐 매니저
- 음성 명령 인식 모듈
- 라이브 채팅 모듈

## 시나리오 3: 1:1 전환 (골드 챗)

### 흐름

1. 멘티가 50,000원 골드 챗 결제
2. 골드 챗 권한으로 1:1 비공개 음성 멘토링 요청 발생
3. 멘토 화면에 1:1 전환 알림 표시
4. 멘토가 음성 명령으로 수락 ("카풀링, 1:1 수락")
5. 라이브 채팅이 임시 일시정지되고 1:1 비공개 채널 개설
6. 비공개 멘토링 5~10분 진행
7. 종료 후 메인 세션 복귀

### 관련 모듈

- 질문 큐 매니저 (1:1 권한 관리)
- 음성 명령 인식 모듈
- WebRTC SFU 서버 (구현 외, 인터페이스로 연동 예정)

## 통합 데모 시연 흐름

발표 시 데모는 시나리오 2를 중심으로 진행한다. 세 모듈이 어떻게 유기적으로 연결되는지 가장 잘 보여주기 때문이다.

1. 멘토 화면과 멘티 화면을 분할 디스플레이로 표시
2. 멘티 화면에서 채팅 입력 및 슈퍼챗 결제 (모킹)
3. 질문 큐에 슬라이드 컨텍스트 포함 등록 확인
4. 멘토 화면에서 음성 명령으로 다음 질문 호출
5. TTS 출력 및 응답 후 채팅 상태 변화 확인
